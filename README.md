# SignatureCat — Landing page

Static marketing site for **SignatureCat** (https://signature.cat), hosted on **GitHub Pages**.
Centrally manage Gmail signatures across a Google Workspace.

## Tech & principles

- **Static + a tiny zero-dep build** - plain HTML + one CSS file + two small JS files, plus `node build.mjs` which pre-renders one indexable page per locale. No framework, no runtime CDN, no npm dependencies.
- **System fonts** — zero webfont downloads (perf-first, per design system).
- **Design system** — warm minimalism + glassmorphism (ref: tryklack.com). Cream background, achromatic, single pink accent (`#f2a8ff`). Light + Dark via `prefers-color-scheme`. See space `01_design-system.md`.
- **Accessibility** — WCAG AA contrast, semantic HTML, focus states, `prefers-reduced-motion` respected.
- **SEO** - one crawlable, self-canonical page **per locale** (`/`, `/pl/`, `/de/`, `/fr/`) with reciprocal `hreflang` (+ `x-default`), localized `<title>`/description/Open Graph/`og:locale`/JSON-LD, `sitemap.xml`, `robots.txt`. Browser-language routing is server-side (Cloudflare Worker), never a client redirect.

## Internationalization & SEO routing

Each locale is its **own indexable URL**, pre-rendered from the single source:
`/` (en, x-default), `/pl/`, `/de/`, `/fr/`. This is what lets search engines
index every language (the old single-URL setup could not).

- **Source of truth:** structure in `index.html` (`data-i18n` keys), copy in
  `assets/js/i18n.js`. `build.mjs` applies the dictionary to produce each
  locale's page + per-page `<title>`/description/canonical/hreflang/og/JSON-LD,
  fills the JS-only `data-i18n` placeholders (so crawlers see real content), and
  rewrites asset refs to root-absolute (`/assets/...`) so `/pl/` resolves them.
- **Run after any edit to `index.html` or `i18n.js`:** `node build.mjs` (idempotent;
  it asserts so). Commit the regenerated `/`, `/pl/`, `/de/`, `/fr/`, `sitemap.xml`.
  The `/pl/` etc. files are generated - never edit them by hand.
- **Client (`app.js`):** locale comes from the **URL path** (the page it was
  served), so the client matches the pre-rendered HTML. The language switcher
  sets a `sigcat_locale` cookie (manual override) and navigates to `/pl/` etc.
- **Browser-language redirect:** done **server-side** by the Cloudflare Worker
  (`cloudflare/`), not a client redirect (which would risk SEO). On `/` it reads
  the cookie (manual choice wins) else `Accept-Language` and 302s to the locale;
  English/crawlers stay on `/`. See `cloudflare/README.md`. Until the Worker is
  deployed the per-locale pages + `hreflang` already work; only the auto-redirect
  is absent.
- Proper names are never translated: SignatureCat, Google, Workspace, Gmail, Stripe, GCP, Secret Manager, Directory API, Domain-Wide Delegation, RODO/GDPR.
- **No "AI-tell" typography** in copy: plain ASCII `-`, no em/en dashes or invisible characters (locale curly quotes `„ "`, `« »` are fine - correct per-language typography).

## Pricing calculator

Mirrors the **graduated** per-seat model from `signaturecat/app` → `docs/06_stripe_billing.md`:

| Workspace user count | Price per user / month |
|---|---|
| 1 user | Free (no Stripe subscription) |
| 2 – 50 | $0.80 |
| 51 – 120 | $0.70 |
| 121+ | $0.60 |

The calculator applies a **single flat tier rate to the whole headcount** (rate for the size band × number of users) — it does NOT sum seats across tiers. Examples: 50 users = $40.00 (0.80×50), 80 users = $56.00 (0.70×80), 121 users = $72.60 (0.60×121), 1 user = Free. It shows `N users × $rate / user / mo` and the monthly total. Currency via `Intl.NumberFormat(locale, { currency: 'USD' })`. Totals are estimates excluding tax.

> Source of truth: `rateForCount()` / `computeTotal()` in `assets/js/app.js`. If tiers change, update those **and** the tier table + this README.
> NOTE: this flat-per-tier estimate intentionally differs from `signaturecat/app` → `docs/06_stripe_billing.md` (graduated). Reconcile if the landing estimate must match real Stripe invoices.

## Deployment (GitHub Pages)

1. **Run `node build.mjs` and commit** the regenerated `/`, `/pl/`, `/de/`, `/fr/`, `sitemap.xml`. (CI does not build; the committed output is what ships. A future GitHub Action could run it as a sync-check.)
2. Repo **Settings → Pages → Build and deployment → Source: Deploy from a branch**, branch `main`, folder `/ (root)`.
3. Custom domain `signature.cat` is set via the `CNAME` file. DNS/infra is owned by the **DevOps Team**.
4. `.nojekyll` ensures directories/assets are served verbatim.

## Server-side language redirect (Cloudflare Worker)

The browser-language redirect (with manual cookie override) is a **Cloudflare
Worker** on `signature.cat/*`, in front of this static origin - see
[`cloudflare/README.md`](cloudflare/README.md). The Worker code is owned here
(frontend); the Cloudflare zone + route + DNS are owned by **DevOps**. The site
is fully functional without it (just no auto-redirect).
