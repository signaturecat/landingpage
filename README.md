# SignatureCat — Landing page

Static marketing site for **SignatureCat** (https://signature.cat), hosted on **GitHub Pages**.
Centrally manage Gmail signatures across a Google Workspace.

## Tech & principles

- **Static + a tiny zero-dep build** - plain HTML + one CSS file + two small JS files, plus `node build.mjs` which pre-renders one indexable page per locale. No framework, no runtime CDN, no npm dependencies.
- **System fonts** — zero webfont downloads (perf-first, per design system).
- **Design system** — warm minimalism + glassmorphism (ref: tryklack.com). Cream background, achromatic, single pink accent (`#f2a8ff`). Light + Dark via `prefers-color-scheme`. See space `01_design-system.md`.
- **Accessibility** — WCAG AA contrast, semantic HTML, focus states, `prefers-reduced-motion` respected.
- **SEO** - one crawlable, self-canonical page **per locale** (`/`, `/pl`, `/de`, `/fr`; canonical URLs carry no trailing slash - the edge Worker 301s slashed requests) with reciprocal `hreflang` (+ `x-default`), localized `<title>`/description/Open Graph/`og:locale`/JSON-LD, `sitemap.xml`, `robots.txt`. Browser-language routing is server-side (Cloudflare Worker), never a client redirect.

## Internationalization & SEO routing

Each locale is its **own indexable URL**, pre-rendered from the single source:
`/` (en, x-default), `/pl`, `/de`, `/fr`. This is what lets search engines
index every language (the old single-URL setup could not).

- **Source of truth:** structure in `index.html` (`data-i18n` keys), copy in
  `assets/js/i18n.js`. `build.mjs` applies the dictionary to produce each
  locale's page + per-page `<title>`/description/canonical/hreflang/og/JSON-LD,
  fills the JS-only `data-i18n` placeholders (so crawlers see real content), and
  rewrites asset refs to root-absolute (`/assets/...`) so `/pl/` resolves them.
- **Run after any edit to `index.html` or `i18n.js`:** `node build.mjs` (idempotent;
  it asserts so). Commit the regenerated `/`, `/pl/`, `/de/`, `/fr/`, `sitemap.xml`.
  The `/pl/` etc. files are generated - never edit them by hand.
- **Client (`app.js`):** locale comes from the **URL path** (the page it was
  served), so the client matches the pre-rendered HTML. The language switcher is
  **crawlable `<a href="/pl/">` links** (work without JS); on click, JS records a
  `sigcat_locale` cookie (manual override) and the `<a>` handles navigation.
- **Browser-language redirect:** done **server-side** by the Cloudflare Worker
  (`cloudflare/`), not a client redirect (which would risk SEO). On `/` it reads
  the cookie (manual choice wins) else `Accept-Language` and 302s to the locale;
  English/crawlers stay on `/`. The Worker also sets `Content-Language` per
  locale on HTML responses. See `cloudflare/README.md`. Until the Worker is
  deployed the per-locale pages + `hreflang` already work; only the auto-redirect
  + `Content-Language` are absent.
- Proper names are never translated: SignatureCat, Google, Workspace, Gmail, Stripe, GCP, Secret Manager, Directory API, Domain-Wide Delegation, RODO/GDPR.
- **No "AI-tell" typography** in copy: plain ASCII `-`, no em/en dashes or invisible characters (locale curly quotes `„ "`, `« »` are fine - correct per-language typography).

## Legal pages (/legal, terms, policy)

Generated by `build-legal.mjs` (zero dependencies, same philosophy as
`build.mjs`) from the Markdown sources in `legal/src/`:

| URL | Content | Indexing |
|---|---|---|
| `/legal` | hub linking every language version + DPA info | index, self-canonical, in sitemap |
| `/pl/terms`, `/pl/policy` | Polish Terms / Privacy Policy (legally binding) | index, self-canonical, in sitemap |
| `/en|de|fr/terms`, `/en|de|fr/policy` | automatic translations (each opens with a supremacy banner linking the Polish version) | `noindex,follow` |
| `/terms`, `/privacy` | meta-refresh redirect stubs to the Polish versions (keep the banner/footer short URLs working) | `noindex` |

SEO decisions: the EN/DE/FR legal pages are machine translations, so they are
`noindex,follow` (avoids scaled-content/duplicate signals); because the
alternates are not indexable, legal pages intentionally carry NO hreflang
cluster. Only `/legal/`, `/pl/terms/` and `/pl/policy/` are added to
`sitemap.xml` (the `LEGAL_URLS` list in `build.mjs`).

The DPA is not published as a page - it is concluded in English only, on
request via contact@signature.cat (the sub-processor list is confidential
under the DPA, which is why the footer has no public sub-processors page).

Update flow: edit the source of truth (workspace `legal/` documents, owned by
the legal workstream), copy the changed files into `legal/src/`, then run:

```bash
node build-legal.mjs
```

and commit `legal/src/` + the regenerated page directories. Styling lives in
`assets/css/legal.css` (uses the tokens from `style.css`, Light/Dark, WCAG AA).

## Pricing calculator

Mirrors the **graduated** per-seat model from `signaturecat/app` → `docs/06_stripe_billing.md`:

| Workspace user count | Price per user / month |
|---|---|
| 1 - 50 | $0.80 |
| 51 - 120 | $0.70 |
| 121 - 300 | $0.60 |
| 301+ | $0.55 |

No free tier: every Workspace starts with a **7-day free trial** (card collected at sign-up, charged when the trial ends), then pays per active user from the 1st seat.

The calculator applies a **single flat tier rate to the whole headcount** (rate for the size band × number of users) — it does NOT sum seats across tiers. Examples: 1 user = $0.80 (0.80×1), 50 users = $40.00 (0.80×50), 80 users = $56.00 (0.70×80), 121 users = $72.60 (0.60×121), 400 users = $220.00 (0.55×400). It shows `N users × $rate / user / mo` and the monthly total. Currency via `Intl.NumberFormat(locale, { currency: 'USD' })`. Totals are estimates excluding tax.

> Source of truth: `rateForCount()` / `computeTotal()` in `assets/js/app.js`. If tiers change, update those **and** the tier table + this README.
> NOTE: this flat-per-tier estimate intentionally differs from `signaturecat/app` → `docs/06_stripe_billing.md` (graduated). Reconcile if the landing estimate must match real Stripe invoices.

## Deployment (GitHub Pages)

1. **Run `node build.mjs` and commit** the regenerated `/`, `/pl/`, `/de/`, `/fr/`, `sitemap.xml`. (CI does not build; the committed output is what ships. A future GitHub Action could run it as a sync-check.)
2. Repo **Settings → Pages → Build and deployment → Source: Deploy from a branch**, branch `main`, folder `/ (root)`.
3. Custom domain `signature.cat` is set via the `CNAME` file. DNS/infra is owned by the **DevOps Team**.
4. `.nojekyll` ensures directories/assets are served verbatim.

## Edge Worker: language redirect, security headers, cookie consent

A single **Cloudflare Worker** on `signature.cat/*` (in front of this static
origin) provides: the browser-language redirect (with manual cookie override),
**security headers incl. an ENFORCED CSP with per-request script nonces**, and
the injected **cookie consent banner** (necessary always-on; analytics opt-in;
re-opened by the `.js-cookie-settings` footer links; consent stored 12 months
in `sigcat_consent`) - see [`cloudflare/README.md`](cloudflare/README.md).
The Worker code is owned here (frontend) and **auto-deploys on merge to
`main` via Cloudflare Workers Builds** (dashboard-side Git integration);
the Cloudflare zone + route + DNS are owned by **DevOps**. Without the Worker
the site still serves (no auto-redirect, no headers, no banner).

> **CSP WARNING:** any new external resource (script, font, iframe, image
> host, fetch/XHR target) is BLOCKED until added to `buildCsp()` in
> `cloudflare/worker.js` and merged to `main` (the Worker auto-deploys via
> Cloudflare Workers Builds - see `cloudflare/README.md`; manual `wrangler
> deploy` is emergency-only and overrides Git builds). Google Analytics hosts
> are already allowlisted; the GA loader must run only when
> `window.sigcatConsent.analytics === true`.
